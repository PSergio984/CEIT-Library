# Plan 02-03: Permanent QR Replay Mitigation

This plan mitigates QR code replay attacks by introducing short-lived dynamic timestamps, client-side auto-refresh polling, server-side clock drift checks, and nonce tracking.

<threat_model>
| Threat | Classification | Mitigation |
| :--- | :--- | :--- |
| Spoofing | S | QR code decryption using App crypt key ensures only authentic client-generated codes are accepted. |
| Tampering | T | SHA256 HMAC signature covering user_id, nonce, and timestamp verifies integrity and blocks parameter modification. |
| Repudiation | R | Storing the scanned nonce in a memory cache lock blocks reuse, preventing unauthorized replay of a prior check-in. |
| Information Disclosure | I | Short-lived QR payload minimizes window of exposure if a student screenshot is intercepted. |
| Denial of Service | D | Nonce cache tracking uses a lightweight transient memory store with a fast 150-second TTL. |
| Elevation of Privilege | E | Enforcing ±60s clock drift and 30s freshness validation ensures only physically present, real-time requests succeed. |
</threat_model>
